      1101. Intent.kt
        1103. Read-only interface for an intent node; the primary type consumed by all rendering and command layers

          1107. Core interface: a node in the intent tree with identity, text, tree linkage, typed field schema, field values, and timestamps

          1119. fieldValues(): Map<String, Any> — current field value map keyed by field name
          1121. isMeta(): Boolean — true for system/type/macro entities hidden from end users
          1123. participantIds(): List<Long> — raw participant IDs from the underlying Relationship op
          1125. createdTimestamp(): Long? — epoch-nanos creation time, null if not recorded
          1127. lastUpdatedTimestamp(): Long? — epoch-nanos of most recent mutation, null if not recorded
      1129. IntentImpl.kt
        1131. Mutable internal implementation of Intent; instances live in VoluntasIntentService.byId and are updated in-place during stream replay
        1133. MutableIntent
          1135. Concrete mutable implementation of Intent used exclusively by the backend; all fields are var and updated as ops are replayed
          1137. All Intent interface methods — delegate to mutable backing properties
      1139. FocalScope.kt
        1141. Immutable snapshot of navigation context returned by getFocalScope; consumed by all display layers
        1143. FocalScope
          1145. Data class holding the focused intent, its full ancestor chain back to root, and its direct children
          1147. focus: Intent — the currently focused intent
          1149. ancestry: List<Intent> — parent chain from focus up to (not including) root, nearest-first
          1151. children: List<Intent> — direct children of the focused intent
      1153. CommandResult.kt
        1155. Wraps the outcome of any command execution; carries a message, optional navigation change, and optional new entity ID
        1157. CommandResult
          1159. Data class returned by every command; clients use newFocalIntent to update displayed focus and id to locate newly created entities
          1161. message: String — human-readable result or error description
          1163. newFocalIntent: Intent? — updated focal intent after navigation-changing commands (focus, up, add)
          1165. id: Long? — ID of any newly created entity (e.g. after addIntent)
      1167. IntentExt.kt
        1169. Kotlin extension functions on Intent; adds utility behaviour without modifying the interface
        1171. Intent.getAncestry(): List<Intent> — walks parent chain to root; returns ancestors ordered nearest-first
      1173. IntentService.kt
        1175. Full read-write service interface combining mutation, query, and persistence; implemented by VoluntasIntentService
        1177. IntentService
          1179. Interface extending IntentStateProvider with mutation and persistence operations
          1181. addIntent(text: String, parentId: Long): Intent — creates a plain STRING_INTENT_TYPE intent
          1183. edit(id: Long, newText: String): CommandResult — updates the text of an existing intent
          1185. moveParent(id: Long, newParentId: Long): CommandResult — re-parents an intent to a new parent
          1187. getAll(): List<Intent> — all non-meta user-visible intents
          1189. writeToFile(fileName: String) — persists the current op list to a .pb stream file
      1191. IntentStateProvider.kt
        1193. Read-only sub-interface passed to web sessions and display layers; limits access to query operations only
        1195. IntentStateProvider
          1197. Minimal query interface: just enough to render the tree and navigate without mutation access
          1199. getById(id: Long): Intent? — look up any intent by its entity ID
          1201. getFocalScope(id: Long): FocalScope — get focus + ancestry + children for the given ID
      1203. IntentStreamConsumer.kt
        1205. Bridges the gRPC SubmitOp RPC to the service; all high-level mutations are expressed as SubmitOpRequest
        1207. IntentStreamConsumer
          1209. Single-method interface: accepts a SubmitOpRequest oneof and routes it to the appropriate service operation
